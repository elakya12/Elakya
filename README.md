# Elakya
# 🤖 Self-Learning AI Engineering with AI

> A personal knowledge base for learning AI engineering concepts — built alongside AI, powered by curiosity.

---

## What this is

This repository is a structured self-study log for learning AI engineering. Every concept, experiment, and project here was explored with the help of AI tools (Claude, ChatGPT, Gemini, etc.) as both a learning partner and a subject of study.

The goal: understand how modern AI systems are built — from embeddings and fine-tuning to RAG pipelines and LLM deployment — by building and breaking things hands-on.

---

## Learning areas

| Area | Topics covered |
|------|---------------|
| **Foundations** | Transformers, attention, tokenization, embeddings |
| **Prompt engineering** | Few-shot, chain-of-thought, structured outputs, tool use |
| **RAG** | Vector databases, chunking strategies, retrieval evaluation |
| **Fine-tuning** | LoRA/QLoRA, dataset prep, PEFT, evaluation |
| **LLM APIs** | OpenAI, Anthropic, Gemini — using SDKs and REST |
| **Agents & tools** | Tool calling, memory, multi-agent systems, MCP |
| **Deployment** | Quantization, serving (vLLM, Ollama), latency tuning |
| **Evaluation** | LLM-as-judge, benchmarks, hallucination detection |

---

## How I learn with AI

1. **Ask, don't just read** — I prompt AI to explain concepts, quiz me, and challenge my assumptions instead of passively reading docs.
2. **Build while learning** — every topic has a small notebook or script to make it concrete.
3. **Log confusion** — when something doesn't click, I document the question and the eventual answer.
4. **Use AI to review AI output** — I cross-check explanations across models and look for disagreements.
